## Context

当前 WindowsFormTool 仅支持 TSK 合并与通用 INK 规则，缺少将 DPAT CSV 测试数据回写到 TSK 的功能。CSV 格式参考 `docs/Csv格式.csv`，包含 `Test Name` 行与数据区（含 X/Y、Bin、测试项列）。需求是在 UI 下拉中新增 “DPAT INK”，支持最多 25 个 TSK/CSV 批量处理，并根据统计公式生成新的上下限后对异常点 INK。

## Goals / Non-Goals

**Goals:**
- 在下拉功能中新增 DPAT INK，弹窗输入 testName、上下限、sigma、Ink Bin
- 支持多测试项选择与配置列表，第二次选择时可回显已选项配置
- 解析 CSV 并匹配 TSK（CSV 文件名包含 TSK 文件名）
- 仅对 Bin=1 的 pass die 参与统计，计算新上下限并标记越界点
- 将越界坐标写回 TSK（Fail + Bin=Ink Bin），输出到 `DPAT_INK`，文件名追加 `_DPAT`

**Non-Goals:**
- 不改变 TSK 文件格式或 TSK_spec_2013 兼容性
- 不实现图形化可视化/报表输出
- 不支持超过 25 个文件的批处理或自动下载/发布流程

## Decisions

1) **UI 交互方式**
   - 选择在主界面下拉中新增 “DPAT INK”，选择后弹出参数窗口。
   - 理由：与现有 “TSK 合并 / TSK INK” 模式一致，降低学习成本。
   - 备选：新增独立窗口或按钮（放弃，交互分散）。

2) **CSV 结构解析**
   - 通过定位 `Test Name` 行得到测试项列；通过 `Site,Serial,Sbin,Bin,X,Y` 行确定数据区起点。
   - 只解析当前选中的 testName 对应列，减少内存占用。
   - 解析使用健壮 CSV 读法（支持空值；数值解析使用 `InvariantCulture`）。
   - 备选：全量加载并构建所有测试项索引（放弃，内存/性能压力大）。

3) **TSK 与 CSV 的匹配**
   - 规则：CSV 文件名（不含路径）包含 TSK 文件名（不含扩展名），忽略大小写。
   - 无匹配或多匹配时在 UI 提示并阻止执行。
   - 备选：要求用户手动一一配对（放弃，操作复杂）。

4) **统计计算与算法选择**
   - 先按输入上下限进行初筛，再计算新上下限。
   - 提供两种算法选择（UI 单选）：
     - 公式1：均值/标准差（新上限 = 均值 + sigma * 标准差；新下限 = 均值 - sigma * 标准差）
     - 公式2：中位数/IQR（专用sigma = (P75 - P25) / 1.35；新上限/下限基于中位数与专用sigma）
   - 默认使用公式1，公式2作为稳健统计备选。
   - 备选：同时计算并取交/并集（放弃，需求未明确且结果不可控）。

5) **回写策略**
   - 对越界坐标：设置 DieAttribute 为 FailDie，Bin 设置为输入的 Ink Bin。
   - 通过 `Tsk`/`DieMatrix` 原有模型更新并保存，确保 TSK 端序与扩展结构兼容。

6) **输出目录与命名**
   - 输出目录：`SavePath/DPAT_INK`（若未配置则使用默认 `D:\New-Tsk\DPAT_INK`）。
   - 文件名：原名追加 `_DPAT` 后缀。
   - 理由：与现有保存路径配置保持一致，便于批量整理。

7) **多测试项配置与处理顺序**
   - UI 提供测试项下拉与“添加/更新”操作，配置完成后进入已选列表并可回显与编辑。
   - 每个测试项保存独立配置（上下限、sigma、公式、Ink Bin），校验规则沿用现有逻辑。
   - 处理时按列表顺序逐项计算并写回；若同一坐标被多个测试项命中，以后处理的配置结果为准。

## Risks / Trade-offs

- [CSV 格式变体] → 若 header 文本不同会解析失败；增加关键字匹配与错误提示
- [测试项值为空/异常值（如 9999）] → 解析为非数值或超限时过滤；在日志中记录被忽略数量
- [坐标不一致或越界] → 写回前检查 X/Y 范围与 DieMatrix 边界
- [样本数过少导致统计不稳定] → 提示样本数不足并允许继续/取消

## Migration Plan

- 纯新增功能，无数据迁移
- 发布后不影响现有 “TSK 合并 / TSK INK” 流程，可按需回滚 UI 入口

## Open Questions

- 默认算法是否必须固定为公式1，或需要记忆上次选择？
- 初筛上下限为空时是否允许直接使用全量数据？
- CSV 内如存在多行 `Test Name` 或多块数据区，是否只处理第一块？
